# Как добавить 3D модель для глобального доступа (PC и мобильные)

## Важно понимать:

Текущая функция загрузки через админ-панель сохраняет модель **только локально** в браузере (IndexedDB). Это означает, что модель будет доступна только на вашем устройстве.

Для того чтобы модель была доступна **везде** (на PC и мобильных устройствах), её нужно добавить в репозиторий GitHub.

## Инструкция по добавлению модели в GitHub:

### Вариант 1: Через GitHub веб-интерфейс

1. Откройте ваш репозиторий на GitHub: https://github.com/Rogday8/techstore
2. Перейдите в папку `models/`
3. Нажмите кнопку **"Add file"** → **"Upload files"**
4. Перетащите файл 3D модели (например, `samsung.fbx` или `samsung.glb`)
5. Добавьте описание коммита (например: "Добавлена 3D модель для Samsung Galaxy S24 Ultra")
6. Нажмите **"Commit changes"**

### Вариант 2: Через Git в терминале

1. Поместите файл 3D модели в папку `models/` вашего проекта
2. Выполните команды:

```bash
git add models/название_файла.glb
git commit -m "Добавлена 3D модель для товара X"
git push
```

### Шаг 3: Обновите данные товара

После добавления файла модели, обновите путь к модели в `script.js`:

Найдите товар в массиве `products` и обновите поле `model3D`:

```javascript
{
    id: 2,
    name: "Samsung Galaxy S24 Ultra",
    // ... другие поля
    has3D: true,
    model3D: "models/samsung.glb"  // Обновите путь к вашей модели
}
```

### Поддерживаемые форматы:

- `.glb` (рекомендуется) - самый оптимизированный формат
- `.gltf` - с дополнительными файлами
- `.fbx` - поддерживается, но менее оптимизирован

### Примечание:

- Файлы должны быть не слишком большими (рекомендуется до 10-20 MB)
- После добавления модели в GitHub, она будет доступна через 1-2 минуты после деплоя
- Модель автоматически загрузится на всех устройствах (PC и мобильные)

## Текущая функция загрузки (локальная):

Функция загрузки через админ-панель всё ещё работает, но сохранённые модели:
- ✅ Доступны только на текущем устройстве
- ❌ Не доступны на других устройствах
- ❌ Не доступны после очистки данных браузера

Используйте её для тестирования, но для продакшена добавляйте модели в GitHub.
